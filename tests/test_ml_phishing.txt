# PhishingShield AI/ML Test Case
# Feature: Neural Network URL Classifier (Client-Side)

## 1. Test Overview
This test verifies that the "Risk Engine" correctly uses the new Heuristic ML Model to classify URLs based on structural features (TLD, special chars, length) rather than just static blacklists.

## 2. Test Vector
**User Simulation URL**:
https://example.com/?ml_test=http://secure-login@paypal-update.xyz

**Decoded Malicious Target**:
`http://secure-login@paypal-update.xyz`

## 3. Threat Analysis (Why it is malicious)
1.  **@ Symbol**: Used to obscure the actual domain. Browsers interpret text before `@` as a username. The real domain is `paypal-update.xyz`, not `paypal.com`.
2.  **Suspicious TLD**: `.xyz` is a high-risk TLD often used by scammers.
3.  **Hyphenation**: `paypal-update` is a "Combosquatting" technique to look like a sub-service.
